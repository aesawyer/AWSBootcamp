In this lab we will be creating a serverless infrastructure. We want to have a Lambda function which grabs the Amazon Resource Number (ARN) for itself.

This information will then be saved to an outputlog.txt file saved within an S3 bucket.

We have already created a role for our Lambda function (ARN: arn:aws:iam::918283516283:role/BootcampLambdaRole) and will create the remaining resources using the below instructions:

  1. Create your S3 bucket using the name "[FirstName]-[LabNumber]-serverless-clilab-s3"

    2. aws s3 mb s3://adam-15-serverless-clilab-s3



  2. Create your Lambda function by setting the name to "[FirstName]-[LabNumber]-clilab-lambda", the runtime to Python3.7, the environment variables to the names of our S3 bucket and Lambda function, and finally uploading a Python function

    3. You can download the Python function here.

    aws lambda create-function \
     --function-name adam-15-clilab-lambda \
     --runtime python3.7 \
     --role arn:aws:iam::918283516283:role/BootcampLambdaRole \
     --code [path/to/file]/Bootcampfunction.py \
     --environment Variables={S3_NAME=adam-15-serverless-clilab-s3}

    The function will automatically pull the ARNs for our Lambda function and output our log file.


  3. Invoke your newly created Lambda function

    4. aws lambda invoke \
     --function-name adam-clilab-lambda



  4. Open your newly created outputlog.txt file to validate completion

    5. aws s3 get-object \
     --bucket s3://adam-serverless-clilab-s3 \
     --key dir/outputlog.txt outputlog.txt
